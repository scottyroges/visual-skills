TL;DR

Give every AI GM a multi-year strategic plan — and make GM skill show up as how accurately they see it.

WhatA layered StrategicPlan (window classification + 5 boards) that sits above the GM's existing reactive, per-event decisions.
WhyToday's GM brain only reacts — a contract expires → evaluate; an offer arrives → evaluate. There is no multi-year roster/cap board, no target lists, no draft/dev feedback loop.
ClosesThe three still-weak gm_capManagement correlations — structurally, not by bolting on more per-decision noise.
Size856-line design · 6 plan components · 8 new GM/owner fields · touches 9 decision sites · ships in 3 sub-phases.
The big idea
“Detailed but mis-perceived.”

Every GM builds an equally detailed plan. Bad GMs mis-perceive cap projections, mis-rank targets, misclassify their contention window, and miss cliff years. Good GMs see the world accurately. GM skill becomes plan-perception quality — the structural mechanism that closes the weak correlations, instead of one more noise dial.

How it works

new vs preserved

The strategic plan is a new top layer. World state is filtered through skill-noised perception into the plan; the plan feeds GM, coach, and owner decisions and derives intent/budget for the existing CompliancePlan. The value model and negotiation engine below stay untouched.

Key decisions

9 locked

Locked in the 2026-05-31 brainstorm — the load-bearing choices the rest of the design rests on.

1
GM skill → plan model
Detailed but mis-perceived
makes skill structural — the mechanism to close the weak correlations, not one more per-decision noise dial.
Rejectedskill scaling plan detail (weak GMs build thinner plans) — real weak GMs plan in full, they're wrong not lazy, and detail-scaling resists validation.
2
Plan structure
5 layered boards + window classifier (6 total)
separable concerns — each board carries its own update cadence and consumers.
3
Board ownership
GM owns all; deputy stats modulate quality
avoids building deputy agents (deferred); reuses the existing scoutingNetwork stat — good GM hires good scouts, compounding.
4
Integration with CompliancePlan
Medium layer — strategic informs intent/budget/targets; compliance keeps levers
preserves the shipped, tuned lever-cost machinery instead of rewriting what already works.
Rejectedfull replacement (discards the tuned levers) and a thin annotation layer (wouldn't shift behavior) — only a medium layer moves intent/budget without regressing the cap work.
5
Option-set gate hardness
Soft prior — dominant default, breakable by personality / owner / opportunity
a dominant default keeps GM behavior realistic without making it rigid.
Rejecteda hard gate where the window strictly dictates the option set — produces robotic GMs that can't pounce on a value opportunity or bend to owner pressure.
6
Classifier shape
4 discrete states + a confidence score
discrete states are legible and testable; confidence carries uncertainty without continuous-state complexity.
7
Non-GM agent visibility
Partial via role-scoped views (coach + owner; no player/rival)
models real-NHL info silos — coach/owner consume slices; no player or rival leakage.
8
Factual vs perceived split
Each board: factual fields + perceived fields
lets validation check factual fields against world state and perceived fields against noise shapes — isolates where skill lives.
9
Perception cadence
Fresh each rebuild (no Bayesian carry-forward)
real GMs re-evaluate faster than the cadence anyway; carry-forward adds state + tuning for no clear benefit.

Rollout

3 sub-phases, gated

Each phase has a hard exit gate — no advancing until the prior phase demonstrably holds the bands.

2m.AStructural core
Scope

capBoard, orgDepthChart, windowClassification (sticky transitions + owner mandate), position budgets. Wires intent + signingBudget to plan-aware computation. Adds coachView() / ownerView(), new GM/Owner fields, and Owner-Brain mandate emission. No target lists yet.

Acceptance gate
  • ✓No regression on Phase 2d / value bands (util 90-100%, over/under = 0)
  • ✓Window distribution realistic: ~7/8/7/10 ±2 across 32 teams
  • ✓Misclassification differentiates by skill (<40 ≥25%, >70 ≤10%)
  • ✓Cliff detection differentiates by capManagement
2m.BBehavior layer
Scope

proScoutingBoard (FA + deadline target lists, Plan A/B/C + reservation prices), pipelineMap (full readiness), all disruption handlers. Wires into FA bidding, extensions, trade generation/evaluation, callup/sendDown. 2h's evaluateExtensionCandidate implemented against the real planner.

Acceptance gate
  • ✓Target-list walkdown fires (Plan A signs elsewhere → bid Plan B)
  • ✓Disruption handlers fire 6–12 per team-season
  • ✓Cliff-year avoidance differentiates by skill
  • ✓No Phase 2f arc-incidence or 2d cap-band regression
2m.CDraft integration + calibration
Scope

draftBoard wired to draft-pick selection (plan-aware, need-adjusted ranking replaces the scout-blind-spot + draftAcumen heuristic). Full multi-seed calibration sweep. Spec-relaxation decision if correlations don't fully close.

Acceptance gate
  • ✓capManagement × dead_cap r < −0.4 (or defensible relaxation)
  • ✓× assets_lost r < −0.3 · × ltir_misses r < −0.3
  • ✓Full Phase 2 acceptance gate green (2d + 2f + 2i + 2h)

Risks

⚠Cap-utilization regression in 2m.A as intent/budget derivation changes compliance behavior
Mitigation: 2m.A's hard exit gate is the existing band — no proceeding to 2m.B until it demonstrably holds.
⚠Replan-cost explosion during burn-in (board rebuilds × 32 teams × 100 seasons)
Mitigation: dirty-flag per board; rebuild only boards whose inputs changed; perception is O(roster), not O(world).
⚠Window classification thrashes — sticky transitions don't stick
Mitigation: unit tests on transition logic; multi-seed steady-state check shows stability over 5+ seasons for stable rosters.
⚠Owner-mandate dominance — window driven entirely by the owner
Mitigation: bounded +25 shift; high-discipline GMs resist; calibration metric flip_rate < 8%.
⚠Disruption-handler conflicts — concurrent events produce inconsistent state
Mitigation: explicit precedence rules; deltas merged on a copy; rollback on conflict detection.
⚠Tuning-surface explosion — many new dials
Mitigation: enumerate all dials upfront; conservative defaults; tune only those that fail validation.
⚠Performance-budget breach (perception × 6 boards × ~12 replans per team-season)
Mitigation: pre-allocated Float32Arrays; <50μs/board × 6 = 300μs/replan; <50μs per game.

Before you approve

reviewer's checklist

What sign-off actually commits to, the one place to look hardest, and what's still open.

▢ What you're approving
A staged rollout, not a big bang. Sign-off greenlights 2m.A (the structural plumbing) first — behind a hard exit gate that is the existing Phase 2d / value-model band. 2m.B (target lists, disruptions) and 2m.C (draft + correlations) don't proceed until the prior phase demonstrably holds. Each phase is independently revertible.
⚠ Scrutinize hardest
The decision-site integration — the spec's own “most regression-prone section.” It rewires 9 GM decision sites (intent/budget derivation, targeted bid sizing, extension anchors, trade-target ID, draft, levers) but keeps every tuned arithmetic core untouched (value.ts, levers.ts, selectBest23, enumerateReliefOptions). Risk is real but bounded — caught by 2m.A's gate before 2m.B adds target lists. See the decision-site map ›
◯ Still open — non-blocking
Explicitly plan-stage inputs, decided during planning — not blockers to approval:
  1. NeedSlot granularity — 13 slots or collapse to 9?
  2. How far to project unfilled depth slots (Y0 only → Y0–Y4)?
  3. Per-position budget defaults (Kerdman 57/33/10) per window.
  4. planDiscipline distribution — wider or tighter?
  5. Owner-mandate persistence — indefinite or TTL?
  6. Cliff-year threshold X% (start 15%).
  7. Same-precedence disruption collisions on one field.
  8. 2m.C spec-relaxation criteria if a correlation lands at −0.32.

Perception model — noise flavors & determinism

Every perceived field has a function of the same shape: perceiveX(truth, skill, deputy?, rng). Three noise flavors:

Continuous numeric

perceived = truth + N(0, σ),   σ = baseSigma × (100 − skill) / 100

Bounded to a valid range; directional bias (e.g. low capManagement → bullish on ceiling growth) adds a mean shift.

Categorical / bucketed

P(perceived_bucket ≠ truth_bucket) = (100 − skill) / k
P(shift 2 buckets)                  = much smaller (~5% mis-band)

Set membership (core players, cliff years)

per true qualifier:    P(included)         = f(skill)
per non-qualifier:     P(falsely included) = g(skill)

Determinism

Each call is seeded by hash(worldSeed, franchiseId, season, day, boardType, fieldKey) — same inputs, same perception. Re-rolling a board mid-season with unchanged world state yields the same perceived board (no “lucky day”). Deputy modulation: effectiveSkill = clamp(gmSkill + scoutingNetworkBoost, 0, 100).

Disruption handlersevent → plan delta

A disruption is an event a handler converts to a PlanDelta applied to a copy of the plan. Kinds: star_injury, target_lost_in_fa, unexpected_offer, prospect_breakout, prospect_bust, rebuild_stall, coach_fired, key_player_trade_request, rival_blockbuster, owner_mandate_change.

Example — target_lost_in_fa: remove the lost target, promote Plan B; redistribute the unspent Plan A reservation (60% → Plan B, 40% → next slot’s Plan A). Example — prospect_breakout (Y2+ → Y0): bump readiness to now, shift graduation counts, upgrade position health, and if a rebuilder sees ≥2 breakouts, re-evaluate the window. Concurrent disruptions merge by precedence: owner-mandate > seasonality > severity-major > severity-minor.

Testing & performance budget

Testing: unit tests per perception function (truth + skill → expected shape), per board rebuild, and per disruption handler; integration tests for the plan→decision pipeline parameterized over GM skill; a multi-seed calibration sweep; a snapshot test on a known seed (window distribution at season 10); and a full cross-system regression check (~2150 tests + Phase 2d/2f/2c gates).

Performance: sim target < 2ms/game. ~1000 replan events per season × <50µs/board × 6 boards = ~300µs/replan, distributing to <50µs per game from planning — achievable with pre-allocated buffers and seeded deterministic perception.
