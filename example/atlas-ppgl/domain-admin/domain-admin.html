What it isThe operator surface: a sidebar-navigated /admin area for correcting tournaments, results, users, golfers, leagues, and picks — plus manual triggers for every sync and rollup the cron normally runs.
Who sees itOnly users whose Better Auth role is admin. Everyone else is redirected to /dashboard by the layout, and rejected with FORBIDDEN by the API.
Big idea
Every admin capability is a tRPC procedure behind adminProcedure — the /admin layout redirect is UX, the middleware is the boundary.

src/app/admin/layout.tsx redirects non-admins, but the real gate is the adminProcedure middleware in src/server/trpc.ts (web-shell), which throws FORBIDDEN unless the session role is admin. Destructive overrides carry service-layer guards — last-admin protection, commissioner reassignment, the GLOBAL league is undeletable, exclusion blocks on locked picks — and pick overrides are written to the commissioner audit log and trigger earnings recalculation.

Internal architecture

Every console page talks tRPC (server pages via serverTRPC, client forms via useTRPC mutations). All admin.* calls pass the adminProcedure gate before the service fans out — to its own repository for reads and overrides, and to neighbor services for syncs, rollups, and password resets.

health.* sits outside this path — its three probes are publicProcedure and query the db / ESPN directly.

In depth

Admin gate & layout src/app/admin

↑ back to cards

layout.tsx is a server component that calls trpc.auth.me() and does if (!me || me.role !== "admin") redirect("/dashboard") before rendering anything. It wraps every /admin route in a sidebar nav: Overview, Tournaments, Users, Golfers, Leagues, Promo Codes, Cron Logs. The Promo Codes link points at /admin/promos, whose pages live in the payments domain — the nav spans both.

page.tsx is the overview dashboard: one admin.stats call renders total users, leagues, picks this season, the currently-live tournament (linking into tournament ops), and the last cron run's success flag and duration — a one-glance health check before digging into any console.

Tournament ops src/app/admin/tournaments

↑ back to cards

The list page filters by season, status, and name, with an EXCLUDED pseudo-status that maps to the excluded boolean filter, and derives a LIVE badge for UPCOMING tournaments whose start date has passed. The detail page loads via admin.tournament and hands off to the TournamentAdmin client component.

TournamentAdmin is the busiest console: an edit form (name, course, dates, purse, estimated purse, status) with a pair of timezone helpers — toInputDateTime / fromInputDateTime — that treat every datetime-local value as US Eastern regardless of the operator's browser timezone; an exclude toggle that surfaces the service's PRECONDITION_FAILED (locked picks exist) as a Force Exclude escalation button; and four sync buttons — Sync Field, Sync Results, Sync Live Standings, and Run Earnings Rollup (hardcoded to GLOBAL_LEAGUE_ID).

TournamentResults renders the per-golfer results table with inline row editing of finish position, earnings, score-to-par, and rounds — typing MC (missed cut) stores position 0. This is the hand-correction path when ESPN's payout data is wrong, and its edits feed directly into what scoring rolls up.

User ops src/app/admin/users

↑ back to cards

The list page searches name/email/username and shows each user's league count and picks this season (two aggregate queries stitched in the repository). The detail page loads the user plus their leagues and season picks, along with upcoming tournaments to power the test-reminder dropdown.

UserDetail is the account-surgery console: toggle role between user and admin (the service blocks self-demotion and demoting/deleting the last admin), send a password reset (routed through Better Auth's requestPasswordReset), send a test pick-reminder email for a chosen upcoming tournament, remove the user from any league, remove any individual pick (via the audited admin.adminRemovePick), and a confirm-gated delete that cascades picks, memberships, season rows, and auth records — refused while the user still commissions any league.

League & golfer ops src/app/admin/leagues

↑ back to cards

League ops: the list filters by type (GLOBAL/PRIVATE), payment status, and name. LeagueAdmin edits any league's settings — name, maxGolferUses, burnStalePicks, showPicksBeforeLock, requireApproval — and reassigns the commissioner from a dropdown of current members. It adds members through a live admin.users search (existing members filtered out), removes any member except the commissioner, and offers a Danger Zone delete that is simply not rendered for the GLOBAL league (the service refuses it server-side too).

Golfer ops is the small sibling under src/app/admin/golfers: a searchable list showing world ranking and events played, and GolferAdmin, a three-field form correcting name, country, and world ranking — the fix-up path when ESPN data comes in wrong or incomplete.

Cron console src/app/admin/cron

↑ back to cards

page.tsx renders the last 20 cron runs from admin.cronLogs: success/error badge, US-Eastern timestamp, duration, and the stored JSON response pretty-printed per run — the primary debugging view when a nightly sync misbehaves.

Admin API src/server

↑ back to cards

routers/admin.ts mounts 35 procedures on adminProcedure, spanning stats, tournaments (+results, field, exclusion, syncs), users, golfers, leagues, picks, promo codes, and cron. The router stays thin — zod input, service call — with one exception: testPickReminder calls accounts' sendTestPickReminder service directly instead of going through services/admin.ts.

services/admin.ts is where the override guards live: existence checks that turn into NOT_FOUND; updateUserRole/deleteUser refuse self-demotion, self-deletion, and touching the last admin (countAdmins); deleteUser also refuses while the user commissions any league (findLeaguesByCommissioner); removeUserFromLeague refuses to remove a commissioner; deleteLeague refuses the GLOBAL league; toggleTournamentExclusion throws PRECONDITION_FAILED when locked picks exist unless force is passed. adminUpdatePick/adminRemovePick wrap the pick change and a commissionerAction audit row (ADMIN_UPDATE_PICK / ADMIN_REMOVE_PICK) in withTransaction, then recalculate the member's season if the tournament is COMPLETED/FINALIZING.

repositories/admin.ts holds the read models — search lists with escaped ilike, aggregate counts (league members, picks this season, events played), and the platform stats query — plus the destructive writes. Its deleteUser, removeUserFromLeague, and deleteLeague run multi-table cascades (picks → season rows → audit rows → memberships → auth rows) inside db.transaction() directly, a deviation from the repo convention of accepting an executor param and letting services own transactions.

Health probes src/server/routers

↑ back to cards

healthRouter exposes three publicProcedure queries: ping (process up), dbCheck (SELECT NOW() through Kysely), and espnCheck (fetches the current season's schedule via espnProvider.getSchedule and reports event count + first/last event names — a real exercise of the ESPN dependency, not a stub).

It's the one router in the codebase that imports db and queries it directly, skipping the service/repository layering — a deliberate pragmatism for probes that must not depend on the layers they're checking. It's also the only public surface in this domain: everything else requires the admin role.

Data it owns

Honestly: none. Admin owns no Prisma models — it is an operator lens over everyone else's tables.

(no owned models)tournament/golfer/cronLog belong to golf-data, user to accounts, league/leagueMember/commissionerAction to leagues, pick to game, promoCode to payments.

What makes this domain unusual is write reach, not ownership: repositories/admin.ts is the one repository that updates and cascade-deletes across nearly every table in the schema.
