A deterministic hockey-league life-sim: generate a world, simulate seasons, let role-based AI run the franchises, and narrate what happens.

WhatA single-player sports RPG where you watch a whole NHL-shaped league live: games, careers, trades, firings, and storylines, all simulated.
ShapeDomain logic in lib/ (7 domains); a Next.js app/ reads it through an in-memory store. No database — state lives in memory for the session.
SpineEverything hangs off one loop: generateLeague → simSeason → (per day) brains decide & the story engine narrates → offseason → repeat.
1
No god-mode AI — four roles each perceive their own (wrong) world.
GMs, coaches, owners, and players each evaluate the league through noisy, skill-scaled perception. A weak GM literally mis-sees talent. This is the heart of lib/brain.
2
Determinism is a hard contract — same seed, same league, same season.
Every random draw comes from a seeded RNG (createRng); nothing calls Math.random(). This is what makes the calibration harness and replays possible.
3
One season loop is the spine the whole codebase hangs off.
If you read one function, read simSeason: it walks the schedule, calls the game engine, fires the brains between games, and ticks the story engine. Section 2 traces it.
4
State is ephemeral and lives in one Zustand store.
There is no persistence layer yet — the generated league lives in lib/store for the session and the UI subscribes to it. Reference data (calibration baselines) is the only thing on disk.

The season spine

runtime

The one runtime flow worth memorizing. store.simSeason() kicks it off; the gold steps repeat for every game and every day across the 82-game season — that inner loop is where the brains and the story engine do their work — then the offseason runs and the whole thing repeats for the next season.

Repeats per game / per day

Brains never mutate the world directly — they return decisions that sim executes. The story engine only observes. That one-way rule keeps the loop deterministic.
