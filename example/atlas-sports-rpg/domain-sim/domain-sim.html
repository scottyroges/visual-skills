Domain

The deterministic game engine and the multi-season loop — the physics of the whole simulation.

OwnsIntra-game hockey resolved shift-by-shift, the between-game and offseason machinery (injuries, cap, contracts, draft, free agency, awards), and the season orchestration that ties it together.
WhyIt transforms world state into outcomes — stats, injuries, events — that every downstream domain reacts to. It is the one place game rules live.
BudgetA full game resolves in <2 ms so thousands of seasons can run for calibration. That performance bar shapes the whole design.
The load-bearing idea
"Data-oriented and deterministic, by construction."

Players live in a struct-of-arrays (RosterSOA — parallel typed arrays), the hot loop pre-allocates its scratch buffers, and every random draw comes from a seeded createRng. Same seed → same game, every time — which is exactly what makes replays and the calibration harness possible.

sim-game lib/sim/sim-game.ts

↑ back to cards

simGame(context) is the per-game entry. It builds the GameState and RNG, pre-allocates scratch buffers and a batch of RNG values, computes team-level offense and pace, then runs three periods of 40 shifts each. If the score is tied it runs 3-on-3 OT and, if needed, a shootout, before finalizeCGResult rolls up per-player stats into a SimResult. The whole call is the first box of the season loop above.

sim-shift lib/sim/sim-shift.ts

↑ back to cards

The hot loop, ~30 seconds of game time per call. It picks lines (the away team disadvantaged by last change), computes a shift context (line offense quality, pace, fatigue and zone-start bias, momentum), updates time-on-ice, then rolls events as probabilities. Every event — a scoring chance and its shot/save resolution, hits, penalties, takeaways, fights, injuries — is a base rate scaled by the relevant attributes. There are no allocations here; that is the <2 ms budget being defended.

sim-season lib/sim/sim-season.ts · sim-multi-season.ts

↑ back to cards

simSeason walks the schedule day by day: it runs each game, reconciles standings and injuries post-game, then runs a daily tick (fatigue recovery, injury advance, chemistry) and fires the brain triggers. sim-multi-season wraps it with the offseason — playoffs, awards, retirements, draft, free agency, development, and the identity season tick — and emits season events to the story engine. The full path is the spine on the atlas.

roster-soa lib/sim/roster-soa.ts · loader.ts

↑ back to cards

Rather than an array of player objects, a team's roster is a struct-of-arrays: one parallel typed array per attribute. The hot loop indexes by player slot, which keeps memory access cache-friendly and avoids per-shift allocation — the single biggest reason a game fits in the time budget. loadRosterSOA converts GeneratedPlayer[] into this layout once and sorts players by role (NHL skaters, NHL goalies, then AHL).

interface RosterSOA {
  shooting:   Float32Array;  // one slot per player
  skating:    Float32Array;
  fatigue:    Float32Array;
  confidence: Float32Array;
  // …30+ skater fields, 10 goalie fields, all parallel
}

contracts/ lib/sim/contracts · cap.ts

↑ back to cards

A ~22-file engine for everything money. computeCapState derives ceiling, committed hit, dead cap, LTIR, and projections; on top of it sit the negotiation surfaces — a triplet engine for three-team trades, an auction for asset ordering, and a bargaining session for salary/term — each backed by comparable-player indices. Entry-level deals, extensions, qualifying offers, and buyouts round it out.

events/ · km/ lib/sim/events · km
