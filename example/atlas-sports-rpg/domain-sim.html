Domain

The deterministic game engine and the multi-season loop — the physics of the whole simulation.

OwnsIntra-game hockey resolved shift-by-shift, the between-game and offseason machinery (injuries, cap, contracts, draft, free agency, awards), and the season orchestration that ties it together.
WhyIt transforms world state into outcomes — stats, injuries, events — that every downstream domain reacts to. It is the one place game rules live.
BudgetA full game resolves in <2 ms so thousands of seasons can run for calibration. That performance bar shapes the whole design.
The load-bearing idea
“Data-oriented and deterministic, by construction.”

Players live in a struct-of-arrays (RosterSOA — parallel typed arrays), the hot loop pre-allocates its scratch buffers, and every random draw comes from a seeded createRng. Same seed → same game, every time — which is exactly what makes replays and the calibration harness possible.

A small hot core (game, shift, roster layout) wrapped by orchestration (season, offseason) and a set of leaf systems (contracts, events, key moments).

How a season runs

sequence

simSeason is the orchestrator. For each game it calls simGame, which drives ~120 simShift calls; post-game reconciles standings and injuries, then the day’s brain triggers fire.

orchestratorbrain (external to sim)
⏱Inside simShift every event is a probability: P(goal) = base × shooter_skill/100 × goalie_factor × zone_bias × PP_bias. There are no allocations in this loop — that is the <2 ms budget being defended shift by shift.

Data it owns

The engine produces outcomes and the per-game state to compute them; it borrows the schema from world and attributes from profiles.

SimResult · SimSeasonResultThe output of a game / season: event stream, player stats, injuries, and final results.
GameState · LineState · PlayerInGameStateIn-memory per-game state: score, penalties, period, on-ice lines, per-player TOI & stats.
RosterSOAThe struct-of-arrays roster — parallel typed arrays for every skater & goalie attribute.
SkaterSeasonStats · GoalieSeasonStatsAccumulated season totals, rolled up from per-game stats.
CapStateComputed salary-cap state: ceiling, committed hit, dead cap, LTIR, projections.

Reads (never owns): League/Game/Player schema from lib/world; SkaterSkills, CoachProfile, personality from lib/profiles.
