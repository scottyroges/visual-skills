Domain

The layer that reads and understands a subject repo — schemas, routers, imports — so the products can ground their output in real code.

OwnsThe parsed-code model: import edges and the module dependency graph, parsed Prisma models, tRPC procedures, and the diffs between two API or schema surfaces.
RolePure analysis. It takes source text (a schema.prisma, a router .ts, any TS/JS file) and a git scope, and returns structured facts — never opinions, never narrative. The TypeScript compiler is its parser of choice.
WhyWithout it, every product would emit a generic file-tree and a raw diff. With it, a Prisma + tRPC repo gets schema- and router-level understanding a generic repo can't.
The load-bearing idea
"Adapters make the products stack-aware."

selectAdapter picks the first adapter whose detect() passes (a prisma/schema.prisma exists → PrismaTrpcAdapter), else falls back to GenericAdapter — which returns nothing. So a Prisma + tRPC repo gets schema- and router-level understanding that any other repo silently does without, with no special-casing in the products themselves.

How a subject repo gets understood

data flow

selectAdapter probes the repo. If prisma/schema.prisma exists, the PrismaTrpcAdapter feeds that schema into parsePrismaModels and each changed src/server/routers/*.ts into parseRouter, then diffs both. Independently, the atlas gather walks every source file through importsOf and hands it to dependencyNeighborhood. Everything bare-package is dropped — only in-repo edges survive.

stack adapters src/adapters/stack-adapter.ts · prisma-trpc.ts · generic.ts

↑ back to cards

StackAdapter is a four-method interface: name, detect(repoRoot), schemaDiff(scope), and apiDiff(scope). selectAdapter(repoRoot, adapters) returns the first adapter whose detect() resolves true, falling back to a fresh GenericAdapter — so selection is just an ordered probe, and the fallback always wins last.

PrismaTrpcAdapter is the one concrete stack adapter. detect checks for prisma/schema.prisma. schemaDiff reads that file at the base and head refs (fileAtRef), parses both with parsePrismaModels, diffs them, and returns a SchemaBlock — or null if nothing changed. apiDiff filters the changed files to src/server/routers/*.ts (excluding _app.ts), parses each with parseRouter, and emits one ApiBlock per router via diffProcedures. It is the seam where every other parser is wired together.

GenericAdapter is deliberately empty: detect always returns true, schemaDiff returns null, apiDiff returns []. A non-Prisma repo therefore gets a clean file-tree-and-diff recap with no broken schema/API sections.

prisma-schema src/prisma-schema.ts

↑ back to cards

parsePrismaModels(schema) regex-matches model X { ... } blocks into a Map<name, PrismaModel>, where each field is { name, type } taken from the first two whitespace-separated tokens of a line (skipping comments and @@ block attributes). It is a pragmatic parser, not a full Prisma grammar — enough to drive a diff and an ERD.

diffModels(before, after) unions the model names and, for each, computes added / removed / kept fields by name; a model that appears only on one side is also flagged. schemaDiffToBlock(diffs) then renders the changed models as a single D2 sql_table ERD with (+ added) / (removed) markers and the changed class — the SchemaBlock the recap shows.

trpc-parse src/trpc-parse.ts

↑ back to cards

parseRouter(source, routerName) builds a TS SourceFile and finds the first router({ ... }) call's object literal. Each property is one procedure; it walks the property's call chain backward — .query / .mutation / .subscription sets the kind, .input(...) captures the raw input expression text, and a trailing xxxProcedure identifier yields the auth (e.g. publicProcedure → public).

The result is an ApiProcedure[] of { name: "router.proc", auth, kind, input }. It is purely syntactic — it reads the builder chain, not the runtime types — which is exactly enough to diff two surfaces.

Key files
trpc-parse.tsTS-AST walk of a router({...}) object into per-procedure { auth, kind, input }.
Key exports
parseRouter()→ ApiProcedure[] — one entry per router property.
Connections
called byPrismaTrpcAdapter apiDiff — before & after for each changed router
called byatlas — gather-atlas parses routers for the API inventory
feedsapi-diff — its output is the input to diffProcedures

imports src/imports.ts

↑ back to cards

importsOf(source, { valueOnly }) collects module specifiers from import, export … from, and dynamic import(...) via the TS AST, deduped. With valueOnly, fully type-only declarations are skipped — they carry no runtime dependency, so they shouldn't count as architectural edges (a mixed import { type T, value } still counts).

exportsOf(source) returns a module's exported binding names — named decls, export { ... }, re-exports, and export default as "default". Together they are the raw material for both the dependency graph and the atlas's per-module inventory.

dep-graph src/dep-graph.ts

↑ back to cards

dependencyNeighborhood(changedPaths, repoRoot) builds the bounded 1-hop import neighborhood around the changed source files: their outgoing imports and the in-repo files that import them. It calls importsOf per file and resolveModule to turn each specifier into a moduleKey — relative specifiers join + normalize; bare packages resolve to null and are dropped as noise. loadAliases reads tsconfig paths/baseUrl so aliased imports (@/...) resolve too (a diagram-aid heuristic, not full module resolution).

Nodes are capped (default 15) by keeping all changed files plus the highest-degree neighbors, with a +N more placeholder for the rest. It emits both a D2 floor and an editable mermaid graph LR — the recap's mechanical where-it-fits diagram. walkSource (the recursive source lister, skipping node_modules/dist/…) backs the importer scan.

api-diff src/api-diff.ts

↑ back to cards

diffProcedures(before, after, title, id) is the pure surface diff. It keys each side by procedure name and compares a signature string of auth | kind | input: a name missing before is added, a name with a changed signature is changed, a name missing after is removed. The result is an ApiBlock carrying only the changed procedures — unchanged ones are dropped so the recap shows just the delta.

It is stack-agnostic — it diffs ApiProcedure[], not tRPC source — so the same diff would serve any future adapter that can produce that shape.

Key files
api-diff.tsName + signature (auth · kind · input) diff into an added/changed/removed ApiBlock.
Key exports
diffProcedures()→ ApiBlock — only the added / changed / removed procedures.
Connections
consumesparseRouter output — the before/after ApiProcedure[]
called byPrismaTrpcAdapter apiDiff, once per changed router

Data it owns

The parsed-code model — the structured facts it extracts from a subject repo's source. It owns these shapes; the products consume them.

import specifiers · the dependency graphPer-file import / export edges from importsOf, resolved to in-repo module keys and assembled into the bounded where-it-fits neighborhood.
PrismaModel · PrismaField · ModelDiffParsed Prisma models (name + typed fields) and the added / removed / kept field diff between two schema versions.
ApiProcedureA parsed tRPC procedure: { name, auth, kind, input }, read syntactically from the router builder chain.
AliasesThe tsconfig paths / baseUrl matchers used to resolve aliased import specifiers to module keys.
the API & schema surface diffsApiBlock (added/changed/removed procedures) and SchemaBlock (changed-model ERD) — the rendered deltas the recap shows.

Reads (never owns): the subject repo's source files and prisma/schema.prisma — pure external input, accessed through git.ts (fileAtRef, changedFiles).

Seams

Source in (via git refs); structured facts and rendered diagram blocks out. The products call these parsers; code-intel calls nothing back.
